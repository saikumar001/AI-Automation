# Twitter Post Best Practices

This reference provides guidance for creating effective tweets.

## Content Guidelines

### Length
- **Ideal**: 70–100 characters for highest engagement (leaves room for likes/retweets without truncation)
- **Maximum**: 280 characters
- **Threads**: If you need more, create a thread; each tweet should be self-contained but part of narrative

### Hook (First Line)
- The first line appears in timeline and notifications — make it pop
- Use emojis, questions, or bold statements
- Examples:
  - "We just open‑sourced our entire stack. 🎉"
  - "Stop scrolling. This changed how I build software."
  - "Thread 🧵: 5 lessons from scaling to 100k users"

### Hashtags
- **Number**: 1–2 is optimal on Twitter; 3 max
- **Placement**: At end of tweet (before @mentions)
- **Mix**:
  - 1 broad (#tech, #startup, #Python)
  - 1 specific (#OpenSource, #SaaS, #DevOps)
- Avoid over‑tagging (looks spammy)
- Check hashtag volume: popular ones get lost; niche ones may not reach audience

### Mentions (@)
- Use to tag people/companies when relevant (increases reach)
- Don't mention random influencers just for visibility
- Place mentions after main content or at end

### Emojis
- 1–3 adds personality; more looks unprofessional
- Use at beginning or end, not mid‑sentence (except for emphasis)
- Common: 🚀 (launch), 🎉 (celebration), 🔥 (awesome), 💡 (insight), 🧵 (thread)

### Links
- Twitter auto‑generates link previews (image + title)
- Use short, clean URLs (bit.ly, t.co masked anyway but cleaner for users)
- Place link before hashtags if you want preview to show; after hashtags is also fine
- **Tip**: Add a call‑to‑action before link: "Read more →"

### Call‑to‑Action (CTA)
- Ask a question to spark replies: "What's your take?"
- Direct engagement: "Retweet if you agree", "Like if you've seen this" (use sparingly; algorithm may boost)
- Clickable: "Link in bio" (if you want to drive traffic elsewhere)

### Threads (Multi‑Tweet Stories)
- Use (1/…), (2/…) at end of each tweet
- First tweet must hook enough to make people click "Show this thread"
- Each tweet should make sense standalone if possible
- Last tweet should have CTA or conclusion
- Can include up to 25 tweets in a thread

## Visual Content

### Images
- **Size**: 1200 x 675 (16:9) for optimal inline display
- **Formats**: JPG, PNG
- **File size**: < 5MB
- **Alt text**: Add for accessibility (and SEO) – describe the image
- **Text overlay**: Minimal; many read text-only; tweet text must convey message

### GIFs
- Use for humor or emphasis (bigger impact than static image)
- Keep short (2–5 seconds loop)
- File size: < 15MB
- Oversized GIFs get converted to video by Twitter

### Videos
- Not currently supported in this skill (could be added)
- Twitter prefers native video over linked YouTube
- Best length: 30–90 seconds; add captions

## Timing (When to Tweet)

**General best times** (based on engagement studies):
- Weekdays 8–10am (people commuting/checking)
- Weekdays 12–2pm (lunch break)
- Weekdays 4–6pm (end of work day)
- **Avoid**: Late nights, weekends (lower B2B engagement)

**But**: Test your own audience. Twitter's algorithm cares about recent engagement. Consistency matters more than perfect timing.

**Frequency**:
- 1–3 tweets per day is sustainable for a brand account
- Batching: write multiple tweets in one session, schedule throughout day
- Don't spam; quality > quantity

## Audience Considerations

- **B2B / Tech**: Professional tone, data‑driven, insights, job posts, product updates
- **B2C**: More casual, emojis, trends, memes (if brand‑appropriate)
- **Founder/Personal**: Authenticity, journey, lessons learned, behind‑the‑scenes

## What to Avoid

- **Engagement bait**: "Like this if you're a founder" — algorithm may demote
- **Duplicate content**: Twitter flags identical tweets; vary wording
- **All‑caps**: Looks like yelling
- **Thread hijacking**: Don't piggyback on trending hashtags with unrelated content
- **Automated DM spam**: Against Twitter rules
- **Excessive hashtags**: 4+ looks like spam
- **Link shorteners that hide destination**: Use clear, trusted domains

## Post Types & Templates

### Product/Feature Launch
```
🚀 We're thrilled to announce [Feature Name]!

[1‑2 sentence value prop]

Try it now: [link]

#Product #Tech #Launch
```

### Insight / Hot Take
```
Most teams waste time on [common practice].

Here's why it's broken and what we do instead:

[3‑point thread]

What would you change? 👇
```

### Job Opening
```
We're hiring! Looking for a [Role] to join our [Location/Remote] team.

[One line about mission]

Stack: [technologies]

Apply here: [link]

#Hiring #DevJobs #RemoteWork
```

### Open Source Announcement
```
We just open‑sourced [Project Name]! 🎉

It solves [problem] with [approach].

Check it out, star it, contribute:

[GitHub link]

#OpenSource #GitHub #DevCommunity
```

### Gratitude / Milestone
```
1000 customers! 🙏

Humbled by the trust. Thank you to our amazing team and users.

Onward to the next milestone 🚂

#Milestone #Grateful #Startup
```

## Engagement Metrics to Track

- **Likes**: General appreciation
- **Retweets**: Amplification (valuable)
- **Replies**: Conversation (great for community)
- **Quote tweets**: Mixed — can be endorsement or criticism; read them
- **Impressions**: How many saw it (hard to control)
- **Engagement rate**: (Likes + RTs + Replies) / Impressions; 1–3% is typical; >5% is strong

---

**Remember**: Twitter moves fast. Post consistently, engage with replies, and iterate based on what resonates. The skill helps you publish reliably; the strategy comes from you.
